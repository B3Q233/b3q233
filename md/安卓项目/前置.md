### 一、基础编程语言与工具

1.  **Kotlin 核心语法**
    -   基础：变量、函数、控制流、空安全（`?`/`!!`/`let`等）、数据类（`data class`）、密封类（`sealed class`）、扩展函数与属性。
    -   进阶：协程（`CoroutineScope`/`launch`/`async`）用于异步处理（如网络请求、音频播放状态更新）、委托（`by`关键字）、高阶函数与 Lambda 表达式。
    -   原因：项目核心模块均用 Kotlin 重写，协程是处理异步逻辑（如后台播放、网络请求）的关键。
2.  **Java 基础**
    -   基本语法、面向对象（继承、多态）、注解、异常处理，因为 Android 框架部分 API 仍基于 Java，且可能涉及 Java 与 Kotlin 混编。
3.  **Android 开发工具链**
    -   Android Studio：熟悉 Gradle 配置（模块依赖、签名配置、构建变体）、Logcat 调试、Profiler 性能分析。
    -   版本控制：Git（分支管理、提交规范，对应项目的 GitHub 协作流程）。

### 二、Android 核心技术

1.  **Android 基础组件**
    -   四大组件：`Activity`（页面生命周期）、`Fragment`（UI 模块化）、`Service`（后台播放音频，如`lib_audio`的后台服务）、`BroadcastReceiver`（系统事件监听，如耳机插拔控制播放）。
    -   布局与 UI：XML 布局、ConstraintLayout、自定义 View（如播放进度条、歌词控件）、主题与样式（`lib_common_ui`的统一 UI 规范）。
2.  **数据存储与网络**
    -   本地存储：`SharedPreferences`（配置信息）、Room 数据库（播放历史、歌单缓存，对应音频模块的本地数据管理）。
    -   网络请求：Retrofit+OkHttp（封装 API 接口，`lib_api`和`lib_network`的核心）、JSON 解析（Gson/Moshi，处理接口返回数据）、HTTPS 与证书配置（音乐接口的安全访问）。
3.  **多媒体处理**
    -   音频播放：`MediaPlayer`或 ExoPlayer（`lib_audio`的核心，处理音乐播放、暂停、进度控制、音频焦点管理）、音频解码格式（MP3/AAC 等）。
    -   视频播放：`VideoView`或 ExoPlayer（`lib_video`模块，处理 MV 播放、全屏切换）。
    -   原因：音乐类应用的核心是音视频处理，需掌握播放状态管理、后台保活（`lib_pullalive`模块）。

### 三、架构设计与设计模式

1.  **模块化架构**
    -   模块划分原则：按功能边界拆分（如基础层 / 核心功能层 / 应用层）、模块间依赖管理（通过接口而非实现依赖，避免循环依赖）。
    -   Gradle 模块配置：`settings.gradle`声明模块、`build.gradle`配置依赖关系（对应项目的多模块结构）。
2.  **分层设计**
    -   UI 层：MVVM 架构（`ViewModel`+`LiveData`/`StateFlow`，处理 UI 与数据的联动，如播放状态更新到界面）。
    -   业务层：单例模式（如`AudioManager`统一管理播放逻辑）、策略模式（不同播放策略的切换）。
    -   数据层：仓库模式（`Repository`，统一管理本地与远程数据，如从网络拉取歌单 + 本地缓存）。
3.  **常用设计模式**
    -   观察者模式（`LiveData`/`Flow`监听数据变化）、工厂模式（创建不同类型的播放器实例）、代理模式（网络请求的拦截与代理）。

### 四、第三方库与工具集成

1.  **基础工具库**
    -   图片加载：Glide 或 Coil（`lib_image_loader`模块，加载歌手头像、专辑封面）。
    -   事件总线：EventBus 或 Kotlin Flow（跨模块通信，如播放状态变化通知 UI）。
    -   依赖注入：Dagger Hilt 或 Koin（管理模块间依赖，如注入`AudioPlayer`实例）。
2.  **特色功能库**
    -   二维码：ZXing 或 ZBar（`lib_qrcode`模块，实现登录扫码）。
    -   分享：友盟分享或原生 ShareSDK（`lib_share`模块，集成微信 / QQ 分享）。
    -   网络状态监听：WorkManager 或自定义广播（处理网络切换时的播放策略）。

### 五、高级特性与优化

1.  **性能优化**
    -   内存管理：避免内存泄漏（如`Handler`弱引用、`Activity`销毁时取消协程）。
    -   UI 优化：列表复用（`RecyclerView`优化，如歌单列表）、过度绘制优化。
    -   启动优化：冷启动流程简化、延迟初始化非核心模块。
2.  **后台保活**
    -   前台服务（`ForegroundService`，`lib_pullalive`模块核心，避免音乐后台播放时被系统杀死）、双进程守护、利用系统广播唤醒进程。
3.  **兼容性处理**
    -   Android 版本适配（如权限动态申请、API 版本差异处理）、不同设备屏幕适配（`dimens`资源适配）。
