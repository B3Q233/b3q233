## 一、 项目基础信息

1.  **项目定位**：基于 Material 3 设计的  非官方客户端，支持无广告播放、离线缓存等功能
2.  **开发语言**：Kotlin（Android 平台首选语言，支持函数式编程、协程等特性）
3.  **开源协议**：GPL-3.0 许可证
4.  **目标平台**：Android 系统（兼容 Android Auto）

## 二、 核心架构模型

### 2.1 整体架构

项目采用 **MVVM（Model-View-ViewModel）** 架构模式，并结合 **模块化设计** 拆分功能，实现职责分离与代码解耦。

-   -   

### 2.2 架构分层设计

| 层级           | 职责                                           | 核心技术 / 组件                       |
| -------------- | ---------------------------------------------- | ------------------------------------- |
| **View**       | 界面展示与用户交互                             | xml + Material 3                      |
| **ViewModel**  | 管理 UI 状态、处理用户行为、桥接 View 与 Model | ViewModel + Kotlin 协程               |
| **Model**      | 数据层，包含本地数据与远程数据                 | Room 数据库 + 网络请求 + 音频处理逻辑 |
| **Repository** | 数据仓库，统一管理本地 / 远程数据来源          | 单例模式 + 协程异步处理               |

## 三、 核心技术框架与工具

### 3.2 异步处理

-   **Kotlin 协程**：处理所有异步任务（网络请求、数据库操作、音频缓存），通过 `lifecycleScope`/`viewModelScope` 绑定 Android 组件生命周期，自动取消任务避免内存泄漏
-   **Flow**：基于协程的冷流，实现数据的异步流式传输（如歌词进度同步、播放状态更新）

### 3.3 网络请求（歌曲核心相关）

1.  **核心能力**：实现 YouTube Music 接口请求、数据解析、音频流获取
2.  技术选型
    -   自定义 HTTP 客户端：处理请求头构造、响应数据解析
    -   代理适配：支持在非 YouTube Music 服务地区通过代理访问
3.  关键功能实现
    -   歌曲搜索 / 专辑 / 播放列表数据抓取与解析
    -   音频流地址获取与缓存
    -   跳过静音、音频归一化、速度 / 音调调节等音频处理

## 四、 数据库设计

### 4.1 核心存储技术

-   **Room 数据库**：Android 官方 ORM 框架，用于本地数据持久化，替代原生 SQLite，简化数据库操作
-   **数据加密**：对用户登录信息、缓存数据等敏感内容进行加密存储

### 4.2 主要数据表设计

| 表名                | 存储内容                  | 核心字段                                       |
| ------------------- | ------------------------- | ---------------------------------------------- |
| `SongEntity`        | 本地缓存歌曲信息          | id、标题、歌手、专辑、封面、音频路径、播放时长 |
| `PlaylistEntity`    | 用户创建 / 收藏的播放列表 | id、名称、封面、歌曲列表 ID、创建时间          |
| `LyricEntity`       | 歌词缓存                  | songId、歌词文本、翻译文本、时间戳映射         |
| `UserSettingEntity` | 用户偏好设置              | 主题模式、播放音质、缓存策略、登录状态         |

### 4.3 缓存策略

-   二级缓存：内存缓存（临时存储播放中的歌曲信息） + 磁盘缓存（Room 存储持久化数据）
-   离线缓存：支持手动下载歌曲 / 专辑，存储为本地音频文件，关联 Room 数据库元信息

## 五、 登录功能实现

1.  **登录方式**：支持 网易云账号登录，模拟 网易云官方登录流程
2.  核心技术
    -   cookie加密存储在 Room 数据库，避免明文泄露
    -   登录状态管理：通过 dataStore  维护全局登录状态，同步到所有页面
3.  六、 MVVM 架构实践细节



-   6.2 ViewModel 层：状态与业务逻辑管理

-   继承 AndroidX `ViewModel` 类，独立于 UI 生命周期，页面销毁时自动清理
-   通过 `Flow`/`StateFlow` 暴露 UI 状态（如播放状态、加载状态、错误信息）
-   核心职责：接收用户操作事件 → 调用 Repository 处理数据 → 通知 View 更新界面

### 6.3 Model 层：数据与业务逻辑封装

-   Repository 模式

    ：统一管理本地（Room）和远程（网络）数据，对 ViewModel 提供透明的数据访问接口

    ```kotlin
    class SongRepository @Inject constructor(
        private val songDao: SongDao,
        private val apiService: ApiService
    ) {
        // 获取本地缓存歌曲
        fun getLocalSongs(): Flow<List<Song>> = songDao.getAllSongs()
        // 搜索远程歌曲并缓存
        suspend fun searchSongs(keyword: String): List<Song> {
            val remoteSongs = apiService.search(keyword)
            songDao.insertAll(remoteSongs)
            return remoteSongs
        }
    }
    ```

-   **数据模型类**：通过 `data class` 定义实体类（如 `Song`、`Album`、`Lyric`），实现数据序列化 / 反序列化
