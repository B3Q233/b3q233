// 专门用于 UI 层的状态封装
sealed class UiState<out T> {
    data object Idle: UiState<Nothing>() // 初始状态
    data object Loading: UiState<Nothing>() // 加载中
    data class Success<out T>(val data:T): UiState<T>() //成功
    data class Error(val message: String,val code: Int = -1): UiState<Nothing>() //失败
}

首页

~~~
布局
	总体上是可上下滑动的垂直布局
	从上到下分为三部分
	{
		upper_tool_content:统计信息栏，包含两个iconButton，用于显示歌曲的历史
		medium_content:	歌曲快选区域，内部为可左右翻转的列表，用于显示推荐歌曲
		lower_content: 专辑推荐区域，内部为左右翻转的page，用于显示专辑信息
	}
	upper_tool_content区域：
	{
		布局：垂直水平居中布局，中间为两个iconButton
			iconButton
			{
				上下垂直居中布局，上部分为一个圆形的button，下部分为文字区域，两区域水平居中布局	
			}
	}
	medium_content区域
	{
		布局：垂直布局，分为两部分
			{
				上方为标题栏title，靠左
				下方为歌曲列表 recommand_list，靠左
				recommand_list
				{
					内部是一个可左右滑动的list，recommand_list_item 为自定义的View
				}
			}
	}
	upper_tool_content 区域
	{
		可左右翻转的 al_list
			al_list_item 布局如下
			{
				垂直布局
				上方为一个图片用于显示专辑图片
				下方为两行文字，依次换行显示，分别显示专辑名，专辑作者
			}
	}
		
交互
	整个页面能够向下滑动
	整个区域到底之后，向下滑动，会画出一个刷新旋转条 rotation_bar，松开后 rotation_bar 进行旋转动画，
	完成刷新，动画停止
	iconButton 点击导航到其它页面
	recommand_list_item 点击导航至播放页面
	al_list_item 点击导航至专辑页面
	upper_tool_content区域，左右滑动会左右切换al_list，允许显示多个al_list_item
	medium_content区域的 recommand_list 左右滑动会切换 recommand_list，并且每次切换都会只显示一页		recommand_list_item
动画
   页面数据未刷新出来时，进入idle状态，view启动动画，刷新出来后动画停止
